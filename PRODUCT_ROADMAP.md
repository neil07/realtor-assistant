## 一句话定位

**Reel Agent = 房产经纪人的社交媒体助手。**

不是"视频工具"，是"每天帮你维护社交媒体的助手"。经纪人只需要审批和发布，不需要创作。

---

## 我们为谁服务

### 目标用户

关系驱动型经纪人：靠 referral 和 open house 成交，每年 10-25+ 单，知道社交媒体重要，但永远被带看、谈判、文书工作打断，没时间做内容。

**典型画像：Prita**

- 10 年资深经纪人，100% referral 业务
- 有活跃交易，完全没时间研究内容工具
- 第一次用 1.0 时连 @机器人都不会，但看到视频后立刻开始提修改意见
- 问她"要不要出镜拍更精致的视频"，她选"不用，直接帮我生成就好"

**不是我们的用户：Josh 类型**

- 29 岁，Instagram 原生用户，自己就能做内容
- 需要的是 CRM，不是内容工具

### 核心洞察

> 问题不是"不会做内容"，而是"没有启动的契机"。
> 我们要解决的是 **activation**，不是 creation。

---

## 1.0 已验证的事

- 经纪人发照片 → 收到视频，这个流程跑通了
- 对话式修改（"换个音乐"、"加上我的头像"）自然发生
- "即时生成 vs 精心拍摄"，用户选即时生成 — 核心价值成立
- WhatsApp 对非技术用户友好（加好友就能用）

## 1.0 的四个缺口

| 缺口         | 表现                       | 2.0 解法                                  |
| ------------ | -------------------------- | ----------------------------------------- |
| 没有记忆     | 每次从零开始问偏好         | 偏好记忆系统，用过 3 次后首版通过率 ≥ 80% |
| 没有质检     | 视频直接给用户，质量不稳定 | 剧本打分 + 视频检查双重质检               |
| 没有反馈闭环 | 修改意见没有被结构化学习   | 反馈分类 → 偏好更新 → 下次更准            |
| 太低频       | 每月 1-2 次 listing 视频   | 加入每日市场资讯推送                      |

## 2.0 做什么

### 内容矩阵

| 内容类型         | 2.0 范围           | 频率        | 用户体验                           |
| ---------------- | ------------------ | ----------- | ---------------------------------- |
| **新上房源视频** | 打磨到"可直接发布" | 每月 1-2 次 | 发照片 → 收视频 → 反馈 → 改 → 发布 |
| **每日市场资讯** | 新增               | 每天 1 条   | 早上收到图+文案 → 点"发布"或"跳过" |
| Just Sold        | 2.1                | —           | —                                  |
| Brand Story      | 2.2                | —           | —                                  |

**为什么先做这两个：** 新上房源是最高价值场景（关键 48 小时），每日资讯是最高频场景（从每月 1-2 次 → 每天）。Just Sold 频率太低，Brand Story 对上下文要求太高。

### 用户旅程（2.0 完整体验）

```
早上 8:00
├── 收到推送："今日资讯已准备好：《Lehigh Valley 春季购房趋势》"
├── 点 [发布] → 图+文案自动发到 Instagram
└── 或点 [跳过] → 不打扰

某天下午，新 listing 上线
├── 经纪人往群里发了 8 张照片
├── Bot 自动识别是房源照片（不需要 @）
├── Bot 发 WhatsApp 按钮："选个风格？" [活力] [优雅] [专业]
├── 经纪人点 [优雅]
├── Bot 发列表："强调哪些？" ☑厨房 ☑泳池 ☐卧室 ☐花园
├── 经纪人勾选，确认
├── 3-5 分钟后收到视频 + 文案 + hashtag
├── "满意吗？" [直接发布] [调整风格] [重做]
├── 经纪人点 [调整风格] → "音乐换活泼点"
├── 2 分钟后收到新版（只重做了音乐，不重新写剧本）
└── 经纪人点 [直接发布]

下次再发照片时
├── Bot 记住了：这位经纪人喜欢优雅风格、活泼音乐
├── 直接用这个偏好生成，不再问
└── 首版通过率显著提升
```

### 需求收集方式

不是纯对话，是 **对话 + WhatsApp 原生交互组件** 的混合模式：

| 方式          | 用途             | 举例                              |
| ------------- | ---------------- | --------------------------------- |
| WhatsApp 按钮 | 二选一 / 三选一  | "选个风格？" [活力] [优雅] [专业] |
| WhatsApp 列表 | 多选             | "强调哪些房间？" ☑厨房 ☑泳池      |
| 自由文本      | 开放补充         | "还有什么特别说明？"              |
| Web 表单链接  | 复杂定制（beta） | 完整品牌设置页面                  |

Alpha 只用按钮 + 列表（零门槛）。Web 表单链接 beta 再加。

参考体验：YouMind 的对话引出表单模式。

## 分工边界

### OpenClaw vs 后台 Pipeline

|      | OpenClaw（对话层）                 | Pipeline（生产层）               |
| ---- | ---------------------------------- | -------------------------------- |
| 负责 | 跟经纪人沟通，收集需求             | 分析照片，生成内容               |
| 类比 | 餐厅前台：问你吃什么、辣不辣、忌口 | 后厨：看食材、定做法、出菜       |
| 问谁 | 问用户（通过按钮/列表/文本）       | 不问用户，自己从照片和历史里推断 |
| 输出 | ContentRequest（结构化需求单）     | 成品视频/图片 + 文案             |

**一句话：OpenClaw 问人，Pipeline 看图。**

### 经纪人 vs 系统 vs 运营

| 系统自动做                 | 经纪人做          | 运营做             |
| -------------------------- | ----------------- | ------------------ |
| 分析照片、写剧本、生成视频 | 审批/拒绝最终成品 | 处理系统标记的异常 |
| 每日资讯生成               | 选择发布或跳过    | 监控质检模型准确率 |
| 学习偏好                   | 提修改意见        | 新用户 onboarding  |
| 质量检查                   | 发内容到社交媒体  | 调整资讯主题库     |

## 节奏规划

### 2.0 四个阶段（8 周）

| 阶段       | 时间      | 做什么                               | 用户感知                         | 验证什么                     |
| ---------- | --------- | ------------------------------------ | -------------------------------- | ---------------------------- |
| **alpha**  | 第 1-3 周 | 视频质量打磨 + 偏好记忆 + 零摩擦入口 | "视频好了，而且它记住了我的喜好" | 首版通过率、修改轮数         |
| **alpha+** | 第 3-4 周 | 每日市场资讯推送（多格式多平台）     | "每天早上都有内容可以发"         | 推送率、发布率、触达频率     |
| **beta-1** | 第 5-6 周 | 剧本自动打分 + 视频自动质检          | "收到的视频质量更稳定了"         | 质检准确率、异常拦截率       |
| **beta-2** | 第 7-8 周 | 模型适配层 + 渠道抽象                | 用户无感（底层升级）             | 模型切换无影响、新渠道可接入 |

### 2.0 之后

| 版本     | 时间        | 新增内容                               | 新增上下文            | 用户门槛      |
| -------- | ----------- | -------------------------------------- | --------------------- | ------------- |
| **2.1**  | 第 9-12 周  | Just Sold + MLS 自动发现新房源         | MLS listing 数据      | 一次 MLS 授权 |
| **2.2**  | 第 13-16 周 | Brand Story + 一键多平台发布           | 社交媒体账号          | OAuth 授权    |
| **3.0**  | 未来        | CRM 集成 + ROI 追踪 + 全自动           | CRM 联系人 + 成交数据 | CRM API 授权  |
| **远期** | 愿景        | 本地 AI 助手（管邮件、日程、全量数据） | 本地全量数据          | 安装软件      |

### 上下文获取策略

**核心原则：不为拿上下文而拉高门槛。先吃透低门槛上下文。**

```
门槛从低到高：

零门槛（加好友即用）
  ├── 对话历史
  ├── 照片分析（Claude Vision）
  └── 偏好记忆（系统自动学习）
         ↓ 2.0 就做，覆盖 70% 需求

一次授权
  ├── MLS 数据（自动发现新 listing）
  └── 内容日历（系统主动推）
         ↓ 2.1 做

OAuth 授权
  ├── Instagram / Facebook 发布
  └── 效果追踪（点赞、询盘）
         ↓ 2.2 做

深度集成
  ├── CRM（Follow Up Boss 等）
  └── ROI 归因（发布 → 询盘 → 成交）
         ↓ 3.0 做

安装软件（终局形态）
  ├── 邮件、日程、本地文件
  └── 经纪人的全面 AI 助手
         ↓ 当经纪人主动说"能帮我管邮件吗"时做
```

**关于本地部署：** 和合伙人讨论过给经纪人装本地 OpenClaw。结论是——现阶段"加好友"是唯一正确的入口。90% 的上下文可以通过 API 授权获取。本地部署是终局形态，等 3.0 证明价值后，深度用户会主动要求。

## 成功指标

### 必须盯的数字（写进每周复盘）

**视频质量（alpha）：**

| 指标         | 目标            | 1.0 基线 | 怎么量                           |
| ------------ | --------------- | -------- | -------------------------------- |
| 首版通过率   | ≥ 60%           | ~25%     | 用户收到视频后选"直接发布"的比例 |
| 平均修改轮数 | ≤ 2 轮          | 3-4 轮   | 每个 job 的 revision 次数        |
| 偏好命中率   | 第 3 次起 ≥ 80% | 0%       | 首版无风格相关修改的比例         |
| 出片时间     | < 5 分钟        | ~8 分钟  | 从收到照片到发出视频             |
| 单条成本     | ≤ $3.00         | ~$2.65   | API 调用总成本（含质检）         |

**每日资讯（alpha+）：**

| 指标       | 目标      | 怎么量                        |
| ---------- | --------- | ----------------------------- |
| 每日推送率 | 100%      | 每个活跃经纪人每天都收到 1 条 |
| 发布率     | ≥ 40%     | 推了 10 条，至少发 4 条       |
| 周触达频率 | ≥ 5 天/周 | 经纪人与系统有交互的天数      |
| 单条成本   | ≤ $0.10   | 2 次 Haiku 调用 + 图片生成    |

**整体留存：**

| 指标       | 目标      | 怎么量                        |
| ---------- | --------- | ----------------------------- |
| 周活跃留存 | 连续 4 周 | 每周至少 1 次交互的经纪人比例 |

## 风险与应对

| 风险                      | 概率 | 影响           | 应对                                                                |
| ------------------------- | ---- | -------------- | ------------------------------------------------------------------- |
| 经纪人不看每日推送        | 中   | 高频策略失效   | 跟踪前 2 周发布率，<20% 则调整推送时间/内容类型                     |
| 视频质量不稳定            | 中   | 用户流失       | beta-1 质检上线前，人工抽检每条视频                                 |
| IMA Studio API 不稳定     | 低   | 出片延迟       | Ken Burns slideshow 作为视频 fallback，ElevenLabs 作为语音 fallback |
| 10 人邀请制太小看不到趋势 | 中   | 数据不够做决策 | 关注个体行为变化，不做统计显著性要求                                |
| 资讯内容质量（AI 幻觉）   | 中   | 损害信任       | 加免责声明 + 事实类内容走模板不走生成                               |

## 本周行动（The Assignment）

在写任何代码之前，做一件事：

> **找 Prita 再做一次视频生成。这次你什么都不帮。**
>
> 不帮她 @机器人，不解释功能，不代操作。只看。
> 记下她每一次犹豫、每一次没收到回复、每一次问"怎么弄"。

这个观察会直接告诉我们 alpha 阶段的"零摩擦入口"和"WhatsApp 按钮交互"该怎么设计。
