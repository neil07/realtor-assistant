# Smart Info Gap Handling

## Principle

不设单独追问环节。分析照片后直接出故事板，缺失信息标 `[TBD]`，让用户在确认故事板时一起补。

## 信息来源优先级

1. **用户 profile**（`profiles/{phone}.json`）→ 姓名、电话、风格偏好
2. **照片 EXIF / 文件名** → 地址、GPS 坐标
3. **照片内容推断** → 房型、特色、面积估计
4. **标为 [TBD]** → 推断不出的才让用户补

## 必须有的信息（没有会影响视频质量）

| Info     | 推断方式          | 推断不出时            |
| -------- | ----------------- | --------------------- |
| 地址     | EXIF GPS / 文件名 | 故事板标 `[地址 TBD]` |
| 价格     | 无法推断          | 故事板标 `$[TBD]`     |
| 联系方式 | profile 读取      | 故事板 CTA 标 `[TBD]` |

## 可选信息（能推断就用，推断不出跳过）

- 泳池是否加热 → 从照片判断，不确定就不提
- 翻新年份 → 从照片风格推断 "recently updated"
- 景观朝向 → 不确定就不写
- 特色卖点 → 从照片提取，不追问

## 首次用户（在 CONFIRMING 里一起问）

故事板末尾附上：

```
首次合作，顺便帮我确认一下~
• 视频风格：🔥 动感 / ✨ 优雅 / 💼 专业？
• 视频上显示价格吗？
```

回复后存入 profile，以后不再问。

## 缺失信息的提示语气

- 信息全有："看起来齐了，Go? 🚀"
- 缺 1-2 项："帮我补一下价格和地址就能开工了 🙏"
- 缺很多（首次用户）："帮我补一下以下信息，我马上开始~"

## Timeout

- 2 小时无回复 → "Hi~ 还需要做视频吗？补个地址和价格就能开工 📹"
- 24 小时 → 不再跟进，重置到 IDLE
