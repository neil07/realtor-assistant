## 一、系统分层

```
┌─────────────────────────────────────────────────────────────────┐
│                     经纪人（WhatsApp / Telegram）                 │
└───────────────────────────────┬─────────────────────────────────┘
                                │ 消息 / 照片 / 按钮点击
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                    OpenClaw（对话层）                              │
│                                                                  │
│  职责：听懂用户、问必要问题、转发结果                               │
│  记忆：对话历史（最近 N 条，LLM 上下文）                           │
│  不做：生成视频、学习偏好、管理任务状态                             │
│                                                                  │
│  配置文件：SOUL.md / AGENTS.md / SKILL.md                        │
└───────────────────────────────┬─────────────────────────────────┘
                                │ HTTP（双向）
                      ┌─────────┴─────────┐
                      │  3 个核心接口      │
                      │  GET  /api/profile │
                      │  POST /api/generate│
                      │  POST /webhook/    │
                      │       feedback     │
                      └─────────┬─────────┘
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                   Reel Agent 后台（生产层）                        │
│                                                                  │
│  职责：分析照片、生成内容、管理任务状态、每日推送                    │
│  记忆：profile.json（偏好）+ jobs.db（任务状态）                   │
│  不做：跟用户对话、问问题、管消息路由                               │
│                                                                  │
│  组件：dispatcher / job_manager / daily_scheduler /             │
│        feedback_classifier / progress_notifier                  │
└─────────────────────────────────────────────────────────────────┘
                                │
                    ┌───────────┼───────────┐
                    ▼           ▼           ▼
               IMA Studio     ElevenLabs   Claude
              (AI 视频)    (AI 配音)    (剧本/资讯)
```

**一句话原则：OpenClaw 问人，后台记事。**

## 四、场景 C：反馈修改（局部重跑）

```
经纪人                   OpenClaw                    Reel Agent 后台
  │                         │                               │
  │── "音乐换活泼点" ────────>│                               │
  │                         │  ← LLM 识别是对上条视频修改    │
  │                         │── POST /webhook/feedback ────>│
  │                         │   { job_id: "j001",           │
  │                         │     feedback: "音乐换活泼点",  │
  │                         │     agent_phone: "+60xxx" }   │
  │                         │                               │── 分类：music
  │                         │<── { re_run_from: "PRODUCING",│   只重做音乐
  │                         │     new_job_id: "j001-r1" }   │   不重写剧本
  │                         │                               │
  │<── "收到，重新配乐中..." │                               │── 只跑 Step4-5
  │                         │                               │   省 ~60% 成本
  │<── [新视频] ────────────│<── callback "delivered" ─────│
  │                         │                               │── 更新 profile
  │                         │                               │   music_rejected:
  │                         │                               │     ["slow/calm"]
```

**重跑策略（re_run_from 决定从哪步开始）：**

```
修改类型          re_run_from     跳过的步骤         节省成本
─────────────────────────────────────────────────────────
音乐 / 配音       PRODUCING       分析+剧本+分镜      ~60%
风格 / 文案       SCRIPTING       分析                ~20%
照片选取          ANALYZING       无（全流程）          0%
```

## 七、核心承接点

OpenClaw 与后台之间只有 **3 个接口 + 1 个回调**：

```
┌─────────────────────────────────────────────────────────────┐
│  接口                    │  触发时机             │  方向     │
├─────────────────────────────────────────────────────────────┤
│  GET /api/profile/{phone}│ 收到照片时，先查偏好   │  OC → 后台│
│                          │ 有偏好→跳过问题        │           │
│                          │ 无偏好→弹出按钮        │           │
├─────────────────────────────────────────────────────────────┤
│  POST /api/generate      │ 确认参数后，启动生成   │  OC → 后台│
│                          │ 立即返回 job_id        │           │
│                          │ 异步处理（3-5分钟）    │           │
├─────────────────────────────────────────────────────────────┤
│  POST /webhook/feedback  │ 经纪人说"换音乐"等     │  OC → 后台│
│                          │ 触发局部重跑           │           │
├─────────────────────────────────────────────────────────────┤
│  POST (callback_url)     │ 每步完成时推进度       │  后台 → OC│
│  = OpenClaw Gateway API  │ 最终推视频+文案        │           │
│                          │ 每日资讯主动推送        │           │
└─────────────────────────────────────────────────────────────┘
```

## 八、状态机（后台 job 生命周期）

```
                    ┌─────────┐
                    │ QUEUED  │  ← 收到请求，入队
                    └────┬────┘
                         │
                    ┌────▼────┐
                    │ANALYZING│  ← Claude Vision 分析照片
                    └────┬────┘
                         │
                    ┌────▼────┐
                    │SCRIPTING│  ← 写剧本 + 规划分镜（并行）
                    └────┬────┘
                         │
                    ┌────▼────┐
                    │PROMPTING│  ← 生成每个镜头的视觉指令
                    └────┬────┘
                         │
                    ┌────▼────┐
                    │PRODUCING│  ← IMA Studio 生成视频 +
                    └────┬────┘     ElevenLabs 配音（并行）
                         │
                    ┌────▼─────┐
                    │ASSEMBLING│  ← ffmpeg 合成最终视频
                    └────┬─────┘
                         │
               ┌─────────┴─────────┐
          ┌────▼────┐         ┌────▼────┐
          │DELIVERED│         │ FAILED  │
          └─────────┘         └─────────┘
               ↑                   ↑
          经纪人满意           重试 3 次后放弃
               │               → 运营介入
          ┌────▼────┐
          │REVISING │  ← 局部重跑（re_run_from 决定入口）
          └─────────┘
```

## 九、与 1.0 的架构对比

```
                1.0                          2.0
──────────────────────────────────────────────────────────
用户数量        1 个（Prita）               10+ 个经纪人
偏好存储        OpenClaw memory（文本）      后台 profile.json（结构化）
任务管理        无（skill 直接跑完）          SQLite 状态机
生成超时        ❌ 会超时丢失                ✅ 异步队列，不超时
修改成本        全流程重跑（$2.50）           局部重跑（最低 $0.80）
每日推送        ❌ 无法做                    ✅ background scheduler
多语言          ❌ 无                        ✅ 跟 profile 走
品牌定制        ❌ 无                        ✅ 品牌色 + 字体
```
