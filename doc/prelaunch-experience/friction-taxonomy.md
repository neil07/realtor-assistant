# 体验缺陷分类法

> 所有发现都要贴缺陷码。没有缺陷码，就很难稳定排序和复盘。

---

## 严重级别

| 级别 | 定义                                               | 默认处理       |
| ---- | -------------------------------------------------- | -------------- |
| `P0` | 上线前必须解决，否则高概率直接伤激活、信任或口碑   | 停线处理       |
| `P1` | 高优先级体验问题，会显著降低激活率、通过率或复用率 | 上线前尽量解决 |
| `P2` | 中优先级问题，会削弱专业感、成长感、效率感         | 进入首批优化   |
| `P3` | 可优化项，不立刻阻断使用                           | 进入后续迭代   |

---

## 证据标准

每个问题必须至少有一项：

- 原始用户输入
- 原始系统输出
- 页面片段或截图
- 模拟结果包编号

建议额外记录：

- 用户第一反应
- 停顿时长
- 是否需要额外解释
- 是否还愿意继续

## 初始化类

| 代码      | 症状                                | 原则映射   | 默认级别 |
| --------- | ----------------------------------- | ---------- | -------- |
| `INIT-01` | 新用户第一分钟还不知道下一步        | `P5`, `A2` | `P0`     |
| `INIT-02` | 入口像工具说明，不像真人助手        | `P4`, `P5` | `P1`     |
| `INIT-03` | 落地页或欢迎信息 CTA 太多           | `P5`       | `P1`     |
| `INIT-04` | 用户自然起手动作接不住              | `P5`, `A2` | `P1`     |
| `INIT-05` | onboarding 在价值验证前制造额外负担 | `P5`       | `P1`     |
| `INIT-06` | 访谈像审问，不像助手                | `P4`       | `P1`     |

## 信任类

| 代码       | 症状                               | 原则映射   | 默认级别 |
| ---------- | ---------------------------------- | ---------- | -------- |
| `TRUST-01` | 像 spam / bot / 群发，不像可信入口 | `P4`       | `P0`     |
| `TRUST-02` | 用户提出安全疑问后仍得不到明确答复 | `P1`, `P4` | `P0`     |
| `TRUST-03` | 第一条价值主张过度吹嘘，缺真实感   | `P4`       | `P1`     |
| `TRUST-04` | 明显回避价格 / 试用 / 人工支持问题 | `P4`, `A2` | `P1`     |

## 价值验证类

| 代码       | 症状                                   | 原则映射   | 默认级别 |
| ---------- | -------------------------------------- | ---------- | -------- |
| `VALUE-01` | 首次价值验证太慢，用户需要耐心等待解释 | `P5`       | `P1`     |
| `VALUE-02` | 首次视频试用要补太多字段               | `P5`       | `P1`     |
| `VALUE-03` | 资讯首试门槛过高                       | `P5`       | `P1`     |
| `VALUE-04` | 访谈后没有明确 starter task            | `A2`, `P4` | `P1`     |

## 输出形式类

| 代码        | 症状                                   | 原则映射   | 默认级别 |
| ----------- | -------------------------------------- | ---------- | -------- |
| `OUTPUT-01` | 文案、caption、hashtags 明显像 AI 生成 | `P4`       | `P1`     |
| `OUTPUT-02` | 输出夸张、模板化、像 brochure          | `P4`       | `P1`     |
| `OUTPUT-03` | 输出虚构照片里没有的信息               | `P1`, `P4` | `P0`     |
| `OUTPUT-04` | 输出结构混乱，用户不知道怎么看         | `P5`, `A2` | `P1`     |
| `OUTPUT-05` | 交付消息信息层级差，不知道下一步       | `A2`, `P5` | `P1`     |
| `OUTPUT-06` | 资讯太泛，不像本地化内容               | `P4`       | `P1`     |

## 等待与闭环类

| 代码      | 症状                                   | 原则映射   | 默认级别 |
| --------- | -------------------------------------- | ---------- | -------- |
| `FLOW-01` | 等待阶段太沉默                         | `P3`, `A2` | `P1`     |
| `FLOW-02` | 进度提示太技术化                       | `P4`, `P5` | `P2`     |
| `FLOW-03` | 修改请求后不知道系统接下来会干什么     | `A2`       | `P1`     |
| `FLOW-04` | 用户无法轻松 publish / skip / continue | `A2`, `P5` | `P1`     |

## 记忆与成长类

| 代码     | 症状                             | 原则映射   | 默认级别 |
| -------- | -------------------------------- | ---------- | -------- |
| `MEM-01` | returning user 被重复问已知偏好  | `P2`, `P6` | `P1`     |
| `MEM-02` | 第 10 次和第 1 次体验没有差异    | `P2`, `P6` | `P1`     |
| `MEM-03` | 用户感觉不到系统在学习           | `P6`, `A8` | `P2`     |
| `MEM-04` | 资讯和视频两条线彼此不共享上下文 | `P2`, `A3` | `P2`     |

## 运营与后台类

| 代码     | 症状                                   | 原则映射   | 默认级别 |
| -------- | -------------------------------------- | ---------- | -------- |
| `OPS-01` | 仪表盘无法判断用户该走哪条初始化路径   | `A8`, `P3` | `P1`     |
| `OPS-02` | readiness / completeness 不能指导动作  | `A8`, `A2` | `P1`     |
| `OPS-03` | 表单状态和真实跟进状态脱节             | `A8`       | `P2`     |
| `OPS-04` | skill brief 编辑过重，普通运营难以使用 | `P5`, `A5` | `P2`     |

---

## 口碑传播阻断点

以下问题即使功能存在，也会显著阻断口碑传播：

- `INIT-01`
- `TRUST-01`
- `TRUST-02`
- `OUTPUT-03`
- `MEM-01`
- `OPS-01`

原因很简单：

- 朋友不会推荐一个“要解释半天”的产品
- 经纪人不会推荐一个“看起来像 spam”的产品
- 资深经纪人不会推荐一个“会编内容”的产品
- 团队运营不会推荐一个“自己都看不懂谁该怎么跟”的系统

---

## 发现转行动的标准格式

建议每条问题用下面结构记录：

```text
[缺陷码] [级别]
场景：
证据：
用户第一反应：
为什么伤激活/信任/口碑：
建议修法：
```

不要只写“感觉这里不顺”。
