## Track 0：初始化与激活

### 目标

把用户“怎么进来、进来先干什么、会卡在哪里”测透。

### 必测场景

| 场景 ID         | 入口            | 起手动作             | 核心问题                      |
| --------------- | --------------- | -------------------- | ----------------------------- |
| `INIT-E1-A1-01` | 朋友推荐        | 直接发照片           | 是否能最快进入价值验证        |
| `INIT-E1-A2-01` | 朋友推荐        | 先问怎么用           | 系统是否给出单一下一步        |
| `INIT-E1-A3-01` | 朋友推荐        | 先问安全             | 是否足够快建立信任            |
| `INIT-E2-A5-01` | 落地页          | 先填表               | 表单是否挡住首次价值          |
| `INIT-E2-A1-01` | 落地页          | 直接试视频           | CTA 是否足够清晰              |
| `INIT-E2-A4-01` | 落地页          | 先试资讯             | 双主线是否导致犹豫            |
| `INIT-E3-A6-01` | 人工访谈        | 访谈后生成任务       | 访谈是否像助手，不像调研      |
| `INIT-E4-A4-01` | 自然流量消息    | 直接 `daily insight` | 新用户能否绕过复杂 onboarding |
| `INIT-E4-A1-01` | 自然流量消息    | 直接发照片           | 模糊入口能否直接接住          |
| `INIT-E4-A7-01` | 自然流量消息    | 先问价格             | ROI 敏感用户是否被吓跑        |
| `INIT-E2-A8-01` | 落地页          | 只看不发             | 页面是否有唯一主动作          |
| `INIT-E2-A5-02` | onboarding form | 收到表单但不想填     | 是否有“先试后填”的退路        |

## Track 1：首次视频体验

### 目标

测“我敢不敢发”和“这是不是太麻烦”。

### 必测场景

| 场景 ID       | 核心问题                               |
| ------------- | -------------------------------------- |
| `VIDEO-P1-01` | 缺少地址/价格时，补信息是否过多        |
| `VIDEO-P1-02` | 是否要先给 sample，再收偏好            |
| `VIDEO-P1-03` | producing 等待时是否像真人助理         |
| `VIDEO-P1-04` | 交付消息、caption、hashtags 是否够专业 |
| `VIDEO-P1-05` | 修改入口是否像自然对话而不是工单       |

---

## Track 2：首次资讯体验

### 目标

测“这条资讯我今天能不能发”。

### 必测场景

| 场景 ID         | 核心问题                                              |
| --------------- | ----------------------------------------------------- |
| `INSIGHT-P2-01` | 新用户不经复杂 onboarding 能否试到一条资讯            |
| `INSIGHT-P2-02` | 资讯是否过泛、过长、像 AI                             |
| `INSIGHT-P2-03` | publish / skip / shorter / more professional 是否自然 |
| `INSIGHT-P2-04` | 试完资讯后，转视频是否自然                            |

---

## Track 3：从首次到熟悉使用

### 目标

测系统是否真的变成“老搭档”。

### 必测场景

| 场景 ID       | 使用阶段 | 核心问题                       |
| ------------- | -------- | ------------------------------ |
| `LONG-P3-01`  | 第 3 次  | 是否出现路径偏好，提问是否减少 |
| `LONG-P10-01` | 第 10 次 | 是否还在重复问已知偏好         |
| `LONG-P20-01` | 第 20 次 | 是否形成助手心智               |
| `LONG-P20-02` | 第 20 次 | 用户是否愿意转介绍给同事       |

---

## Track 4：运营与后台界面

### 目标

测后台是否真能支撑初始化分流和后续跟进。

### 必测场景

| 场景 ID          | 核心问题                                 |
| ---------------- | ---------------------------------------- |
| `OPS-DASH-01`    | 仪表盘是否一眼看出谁能进视频、谁能进资讯 |
| `OPS-CLIENT-01`  | client detail 是否能指导下一步动作       |
| `OPS-ONBOARD-01` | onboarding 文案是否符合低耐心用户        |
| `OPS-SKILL-01`   | skill brief 编辑是否过重、过内部化       |

---

## 推荐压测顺序

1. 先跑 `Track 0`
2. 再跑 `Track 1` 和 `Track 2`
3. 再跑 `Track 3`
4. 最后跑 `Track 4`

理由：

- 激活没过，后面都白测
- 首次价值不过关，留存没意义
- 长期使用只有在前两步成立后才有意义
- 运营界面要用前面发现的问题来反推是否支撑得住

---

## 每条主线都要回答

### 视频线

- 用户为什么愿意继续发照片？
- 哪一步最容易放弃？
- 第一次交付为什么“敢发”或“不敢发”？

### 资讯线

- 资讯是首用项还是留存项？
- 这条内容为什么“今天能发”或“今天不会发”？

### 访谈线

- 访谈是否显著降低迷茫？
- 访谈后转成任务是否提升推进率？

### 运营线

- 运营有没有足够信号判断用户该走哪条路？

---

## 记录要求

每个场景最少记录：

- `第一反应`
- `最卡的一点`
- `如果这是朋友推荐来的，我会不会继续`
- `如果这是我办公室同事问我，我会不会推荐`
