# 初始化与激活专项打法

> 初始化不是“收集资料”，而是“让迷茫用户在 1 分钟内愿意继续走下去”。

---

## 为什么要单独压初始化

Reel Agent 的真实风险不在“生成后怎么改”，而在更前面：

- 用户是被朋友推荐来的，还是点落地页来的？
- 他是想先发 listing，还是只想先看看 daily content？
- 他是想先试功能，还是先判断你可不可信？
- 他愿不愿意先填表，还是只想被告诉“第一步做什么”？

如果这些问题不单独测，后面的漂亮流程没有意义。

---

## 4 条入口线

### E1 朋友推荐 / 同行转介绍

用户心理：

- “别人说这个能做视频，我来看看”
- “这不会是垃圾信息吧”
- “是不是又要下载一个 app”

重点看：

- 第一条信息是否像真人推荐，而不是工具介绍
- 是否在最早阶段就给出单一下一步
- 是否足够快建立安全感

### E2 落地页进入

用户心理：

- “我还在判断这是什么”
- “视频和资讯都说能做，那我到底先试哪个”
- “我想知道值不值得我花时间”

重点看：

- CTA 是否收敛到一个主动作
- 是否信息过载
- 是否把用户推去填表而不是先拿到价值

> 当前仓库没有完整落地页实现，所以这条以“概念入口”方式评估，不做页面代码 review。

### E3 运营 / 人工访谈导入

用户心理：

- “有人带我上手，我可以先说需求”
- “但别像在审问我”

重点看：

- 访谈后的信息是否真正转成下一步动作
- 是否让用户觉得“被照顾”
- 是否避免把访谈变成新的 onboarding 负担

### E4 直接自然流量消息进入

用户心理：

- “我先发一句试试”
- “我不想理解产品，我想知道它接不接得住我”

重点看：

- 模糊意图是否能被稳稳接住
- 是否无需长解释就能进入首次价值验证

## 8 种起手动作

| 代码 | 用户动作                       | 用户真正想要的                 |
| ---- | ------------------------------ | ------------------------------ |
| `A1` | 先发照片                       | 快速验证“你到底能不能直接做事” |
| `A2` | 先问这是啥 / 怎么用 / 要下载吗 | 先降低认知负担                 |
| `A3` | 先问安全                       | 先判断是否可信                 |
| `A4` | 先试 `daily insight`           | 想先看低成本价值               |
| `A5` | 先填表 / 先完成 onboarding     | 愿意合作，但希望一次搞定       |
| `A6` | 先接受人工访谈                 | 希望有人带，不想自己试错       |
| `A7` | 先问价格                       | 先判断 ROI，再决定是否继续     |
| `A8` | 什么都不发，只看页面或案例     | 先建立心智，不想立刻承诺       |

## 最该压的组合

这 10 组是最容易暴露问题的入口 x 起手动作组合：

1. `E1 + A1`：朋友推荐后直接发照片
2. `E1 + A2`：朋友推荐后先问“怎么用”
3. `E1 + A3`：朋友推荐后先问“安全吗”
4. `E2 + A5`：落地页先进表单
5. `E2 + A1`：落地页看完直接试视频
6. `E2 + A4`：落地页先想试资讯
7. `E3 + A6`：人工访谈后形成任务
8. `E4 + A4`：新用户直接 `daily insight`
9. `E4 + A1`：新用户直接发照片
10. `E4 + A7`：新用户第一句就问价格

## 三条首用路径，不预设谁赢

### Path 1：Video-first

适合：

- 手上有 listing
- 想立刻看结果
- 目标明确

要验证：

- 是不是真的最短路径
- 是否补信息太多
- 是否需要先给示例再收偏好

### Path 2：Insight-first

适合：

- 手上暂时没 listing
- 先想看 daily content 是否值得
- 对“每天可发内容”更有兴趣

要验证：

- 新用户是否能不经复杂 onboarding 就试到一条资讯
- 试完资讯后，转视频是否自然

### Path 3：Interview-first

适合：

- 对工具不熟
- 需要有人带
- 风格、品牌、市场差异大

要验证：

- 访谈是否真的降低负担
- 还是只是多了一层流程

## 访谈转任务：必须压的一层

如果用户说“我不懂这些工具，你直接告诉我第一步做什么”，系统不该继续介绍功能，而该把访谈沉淀成简短、可执行的 starter tasks。

### 要测试的三种表现形式

#### F1 单条助手式建议

示例：

`Next best step: send 6-10 listing photos for your first trial video.`

适用：

- 用户赶时间
- 用户不想看列表

#### F2 轻量任务卡

示例：

- `Task 1: Pick your video style`
- `Task 2: Send one current listing`

适用：

- 用户愿意完成 1-2 个明确动作
- 需要一点结构感

#### F3 分支式推荐

示例：

`Since you don’t have a listing today, I can set up your daily market content first.`

适用：

- 用户没有 listing
- 用户对视频/资讯两者都模糊

### 要测试的任务排序逻辑

| 用户状态               | 推荐顺序                        |
| ---------------------- | ------------------------------- |
| 有 listing、急着发     | 先视频，后资讯                  |
| 没 listing、但在做社媒 | 先资讯，后视频                  |
| 不熟工具、怕麻烦       | 先访谈，后单任务试用            |
| 强品牌型               | 先风格/品牌校准，后视频         |
| 忙碌型                 | 跳过长 onboarding，直接最短试用 |

## 初始化专项评分

除通用评分外，初始化阶段必须额外打 4 个分：

| 维度             | 问题                               |
| ---------------- | ---------------------------------- |
| `entry clarity`  | 用户是否清楚从入口到第一步要做什么 |
| `activation fit` | 路径是否符合用户自然动作           |
| `trust ramp-up`  | 信任建立是否足够快                 |
| `task framing`   | 下一步是否足够短、清楚、像助手     |

### 初始化红线

- 第一分钟后仍不知道下一步：`P0`
- 必须先理解大量功能才能开始：`P1`
- 用户自然起手动作接不住：`P1`
- onboarding 明显挡住首次价值验证：`P1`

---

## 初始化压测时禁止做的事

- 不要替产品解释“其实后面会更简单”
- 不要因为用户是目标用户就默认她会忍耐
- 不要把“用户没耐心”当成用户的问题
- 不要把“以后会做 landing page”当成当前入口设计的免责理由

---

## 初始化压测结束后必须回答的 6 个问题

1. 用户进来后最自然的第一步是什么？
2. 是先视频更顺，还是先资讯更顺？
3. onboarding 应该前置、后置，还是只在部分入口出现？
4. 访谈转任务会减负，还是加负？
5. 用户从哪个入口最容易迷失？
6. 哪个初始化问题最伤口碑传播？

如果这 6 个问题答不出来，说明初始化压测还不够深。
