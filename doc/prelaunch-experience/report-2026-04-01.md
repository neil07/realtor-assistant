## 1. 执行摘要

- 本轮评估时间：2026-04-01
- 评估角色：Codex 全链路走查
- 覆盖 persona：`P1`-`P5`
- 覆盖入口：朋友推荐、自然流量消息、onboarding form、console/operator surfaces、landing-page concept、interview-first concept
- 覆盖场景数：29
- 真实对话场景数：6
- 模拟输出场景数：9 个核心包（`V1 V2 V4 I1 T1 T2 T3 T4 T5`）
- UI/后台场景数：23 个 manual-review 场景

### 一句话结论

当前产品还不适合直接进入 OpenClaw 全链路试点。最主要的问题不是视频功能不存在，而是入口分流、信任问题、refinement/revision 契约、以及 operator 下一步判断这四条链路还没有闭合；其中一部分必须由 backend 修，一部分更适合在 OpenClaw orchestration 层和 console 层一起解决。

## 2. 关键问题的明确回答

### 哪种入口最容易激活

`video-first` 仍然是最容易激活的路径，但前提是用户已经愿意直接发图。真实 HTTP 场景 `INIT-E1-A1-01` 和 `INIT-E4-A1-01` 都能稳定识别为 `listing_video / start_video`，只是当前还多出“先选风格”这一步。

### 哪种入口最容易流失

`trust-first` 最容易流失。`Is this an app?`、`How much per month?`、`How do I know this is secure and not spam?`、`tell me the first step` 目前全部回到同一段 welcome，属于把高敏感意图打平处理。

### onboarding 是否应该前置

不应该。当前 onboarding 文案和 form completion 页面都把“先填表”说成试用前提，这和低耐心、强 ROI 的首用人群不匹配。它应该变成试用后的加速器，或者只在高控制型用户上前置。

### 资讯更适合作为首用项还是留存项

短期看更适合作为“可选首用项”，而不是默认首用项。精确命令 `daily insight` 能起步，但自然 phrasing、refinement、callback payload 还不稳定，说明 insight-first 还没到可无脑推广的程度。

### 访谈转 starter tasks 是否比直接塞表单更适合目标用户

是。mock packs `T2 / T3 / T5` 代表的就是更合理的体验方向，但当前产品没有把这条路径做成真实表面。对不熟工具、怕麻烦的经纪人，starter tasks 明显比 1 分钟表单更轻。

### 哪些问题最伤口碑传播

- trust/pricing/security 问题被打回 welcome
- 系统主动展示的 refinement 命令并不工作
- insight callback 契约当前就有测试失败
- ops 看不出用户该先走视频、资讯还是访谈

## 4. 初始化与激活发现

### 4.1 朋友推荐 / 同行转介绍

发图路径可用，但信任路径不可用。`INIT-E1-A1-01` 能接住照片；`INIT-E1-A2-01` 和 `INIT-E1-A3-01` 都回到大 welcome，导致“这是什么 / 安不安全”的首问没有被直接回答。

### 4.2 落地页进入

仓库里没有可评审的 landing page 实现，所以产品实际上还没有落地页首用路径。当前所有关于 `video-first / insight-first` 的 CTA 对比，都还停留在概念包和 mock pack 层。

### 4.3 onboarding form 进入

这是当前最明显的“先收资料再给价值”。邀请文案、表单标题、完成页 CTA 都把用户推向 setup，而不是 first win。

### 4.4 自然流量消息进入

准确的 canonical phrase 能工作，模糊或自然 phrasing 不稳。`daily insight` 可以，`I do not have a listing today but I want daily content` 会被打到 `property_content`。

### 4.5 访谈转任务

方向对，但还没实现。现有系统没有把 trust-building chat 沉淀成 short starter tasks，只能在 mock pack `T2/T3/T5` 里看到正确形态。

## 5. 首次视频体验发现

### 最打动用户的点

mock pack `V1` 证明视频交付在 tone 上是有希望的，caption 也接近“经纪人会发”的水平。

### 最容易放弃的点

新用户发图后仍被要求先选风格，说明首试还不是最短路径。对于 referral / natural traffic 用户，这一步不是致命 bug，但确实增加了不必要摩擦。

### 最不敢发的原因

一旦脚本滑向 `V2` 那种 brochure 腔，或者修订请求被路由错，用户就会立即怀疑“这是不是 AI 套模板”。

### 修订链路的问题

这是视频线的最大问题。`more professional` 和 `make it more professional` 在 delivered context 下都被识别成 `style_selection`，说明 revision lane 没有稳定压过 style keyword lane。

## 6. 首次资讯体验发现

### 为什么“今天会发”

`I1` 这种本地化 insight mock 是有吸引力的；精确命令 `daily insight` 也能进 lane。

### 为什么“今天不会发”

一旦用户试 refinement，当前体验立刻断裂。再加上 callback payload headline 丢失的测试失败，说明 insight-first 现在还不是闭环产品。

### 资讯转视频的自然度

当前几乎没有。系统没有在 insight lane 成功后自然引出视频试用，也没有在 form completion 后提供 `try today's market content` 这类 CTA。

## 7. 长期使用与成长感知

### 第 3 次

profile 字段会积累，但路径偏好不会显性呈现。

### 第 10 次

用户仍可能在 revision 时被重新拉回 style setup，说明体验并没有明显“变懂我”。

### 第 20 次

console 里没有 path history、activation history、lane confidence，成长感无法被 ops 或用户感知。

### 记忆问题与成长感问题

当前更像“记住了字段”，不像“记住了习惯”。这会限制长期助手心智。

---

## 8. 运营后台与界面问题

### Dashboard

能看 readiness，不能看“推荐路径”。这不够支撑初始化分流。

### Client Detail

能看缺失字段和 Skill brief，但看不到 outcome-oriented next step。缺的是“先让这位用户试什么”。

### onboarding 页面 / 表单

前置 setup 感太强，而且 completion message 只偏向视频。

### skill brief / 运营维护

可以编辑，但过于内部化。对普通运营而言，这更像 prompt 编辑器，不像业务动作台。

## 10. 进入 OpenClaw 全链路前的最小修复清单

### Backend

- 给 `app / security / pricing / first step / insight-first phrasing` 增加明确 intent，不再一律回 welcome
- 把 delivered context 下的自然语言修改优先识别为 revision
- 统一 insight callback payload，兼容 flat 和 v2 content pack
- 如果暂时不支持 `shorter / more professional` refinement，就把这些提示从 response contract 里拿掉

### Console / Frontend

- onboarding 改成 optional accelerator，completion 页面提供 `send first listing` 和 `try today's market content` 的分流 CTA
- dashboard 增加 `recommended path` 与 `next action`
- client detail 把“待完成事项”提升为“建议下一步”，减少纯字段/Skill brief 视角

### OpenClaw Orchestration

- trust-first 用户先给短答，再给一个 starter task，不要直接复读 backend welcome
- 只显示 backend 真正支持的按钮/示例命令
- 把 interview-first 变成真实 starter-task layer，而不是继续推表单

### Shared Contract

- 冻结 `/api/message` 用户可见命令契约
- 冻结 `daily_insight` callback payload schema
- 为“OpenClaw 展示给用户的命令”增加 contract tests，保证 hint 与 behavior 一致
